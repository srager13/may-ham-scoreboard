Websocket Event Types:
- 'score_updated'        // When a hole score is submitted
- 'match_completed'      // When a match finishes
- 'leaderboard_updated'  // When standings change
- 'match_started'        // When a new match begins
- 'hole_completed'       // When all players finish a hole
- 'tournament_updated'   // When tournament details change

Usage Examples:

In Match Screen:

```
import { MatchScreenWithWebSocket } from './components/MatchScreen';

<MatchScreenWithWebSocket 
  matchId="match-uuid" 
  tournamentId="tournament-uuid" 
/>
```

In Leaderboard:

```
import { LeaderboardWithWebSocket } from './components/Leaderboard';

<LeaderboardWithWebSocket tournamentId="tournament-uuid" />
```

Custom Websocket Usage:

```
const { connectionStatus, on, off } = useWebSocket({
  tournamentId: 'abc-123',
  token: authToken,
  autoConnect: true,
});

useEffect(() => {
  const handler = (payload) => {
    console.log('Score updated:', payload);
  };
  
  on('score_updated', handler);
  return () => off('score_updated');
}, [on, off]);
```

Go Backend Integration Points
When a score is submitted in your API:
```
go// After saving score to database
payload := ScoreUpdatedPayload{
    MatchID:    matchID,
    HoleNumber: holeNumber,
    HoleResult: result,
    MatchStatus: status,
}
wsHub.BroadcastToTournament(tournamentID, "score_updated", payload)
```

When leaderboard changes:

```
gowsHub.BroadcastToTournament(tournamentID, "leaderboard_updated", LeaderboardData{
    TournamentID: tournamentID,
    TeamStandings: teamStandings,
    IndividualStandings: individualStandings,
})
```

Testing the WebSocket Connection
You can test with a simple client:
```
javascriptconst ws = new WebSocket('wss://your-api.com/ws/tournaments/abc-123?token=your-token');

ws.onmessage = (event) => {
  console.log('Received:', JSON.parse(event.data));
};

ws.send(JSON.stringify({ type: 'ping' }));
```
